# 측정 전략 로드맵 — 네 방법이 어떻게 서로에게 가치를 넘기는가

> 이 문서는 방법 ①②③④가 **개별 도구가 아니라 하나의 사슬**임을 설명한다.
> 각 방법에서 어떤 결과를 얻고, 어떤 결론에 도달하며, 그것이 다음 단계에 어떤 가치를
> 부여하는지의 **예상 흐름(forecast)** 이다. 개별 명세는 각 `docs/method-*.md` 참조.

**핵심 한 문장**
> 각 방법은 "비용↑·독립성↑"의 사다리이고, 앞 방법은 답을 다 못 내는 대신 **질문을 좁혀서**
> 다음 방법에게 조준점을 넘긴다. ①은 "어디가 의심스러운가", ④는 "얼마나·어느 축인가",
> ②는 "보정계수", ③은 "최종 진실"을 준다.

> ⚠️ 상태 표기: 아래에서 "관측됨"은 실주행으로 확인된 사실, 나머지는 **예상(forecast)** 이다.
> (2026-07 기준: ①만 실주행 검증, ④는 코드 구현·실주행 미착수, ②③ 미착수.)

---

## 사다리 전체 지도

| 방법 | 비용 | 진실 독립성 | 답하는 질문 | 다음에 넘기는 것 |
|---|---|---|---|---|
| **①** 상대(`odom_compare`) | 무료 | 없음 | 융합이 뭘 바꾸나? 건강한가? | "회전이 의심 축" + "진실이 필요함" |
| **④** MCL(`odom_mcl`) | 낮음 | 부분(순환) | 얼마나·어느 축·CW/CCW? | ②에 정량 가설, ③에 기대치 |
| **②** UMBmark(`odom_umbmark`) | 중간 | 완전(끝점) | 계통/랜덤 분리, 보정계수? | 실제 로봇에 밀어넣을 수정값 |
| **③** AprilTag(`odom_apriltag_gt`) | 높음 | 완전(전경로) | 절대 진실은? | 심판 + ④ 재보정 |

## ① 상대 비교 — "의심 축을 지목한다"

**예상 결과** (일부 실측):
- `swerve ≈ fused` 어디서나 (20 m에 <0.2 mm, <0.05°). ← **관측됨**
- `strafe_square`는 drift ≈ 0 (EKF가 x,y에 IMU를 안 씀). ← 예상
- 정사각형 비폐합(핀휠) −4 cm/−4° per loop. ← **관측됨**

**도달 결론:**
1. **융합은 사실상 통과(pass-through)** — 테스트 속도에선 EKF가 바퀴값을 거의 그대로 낸다.
2. **그래서 ①은 절대오차에 구조적으로 눈이 멀었다** — 바퀴 계통오차는 두 추정기에 똑같이
   들어있어 차이로 안 드러난다.
3. 핀휠은 **보였지만 정체불명** — 실행 허용오차 아티팩트(대부분 추정)인지, 진짜 미회전인지,
   odom 오차인지 구분 불가.

**핸드오프**: ①의 진짜 산출물은 숫자가 아니라 **"회전(heading)이 범인 후보"라는 지목**과
**"진실 없이는 안 된다는 증명"**이다. `analyze_method1.py`의 비폐합 분리·CW/CCW 리포트가 이
지목을 정량 형태로 ②에 넘긴다.

---

## ④ MCL 잔차 — "얼마나·어느 축인지 처음으로 외부 기준으로 잰다"

**예상 결과** (forecast):
- 직진 중 잔차가 커졌다가 스캔 보정에서 **급감(snap-back)** — ①의 near-zero와 달리 **처음으로
  odom이 외부 기준에서 벌어지는 것을 본다**. 급감폭 ≈ 보정 사이 누적 odom 오차.
- **`strafe_square`가 ①과 달리 0이 아닐 수 있다** — 라이다가 x,y를 절대 보정하므로, ①이 못 본
  **횡방향 스케일/strafe 슬립**이 여기서 처음 드러난다(병진 오차의 첫 신호).
- CW vs CCW 잔차 부호가 뒤집히면 → **회전 스케일 비대칭**.

**도달 결론:**
- "이 환경에서 운영 위치오차는 ~X cm / Y°" (생산에 직결).
- "odom 드리프트율 ~Z per m·per rev" (단, 순환성 탓 **하한**).
- "**회전 지배 vs 병진 지배**" 판정 (strafe와 square 비교로).

**핵심 역설**: ④는 **위치는 잘 보지만 yaw는 약하다**(MCL yaw는 라이다·맵 특성상 부정확). 그런데
①이 지목한 범인이 바로 **yaw/회전**이다. 즉 **④조차 정작 의심 축(yaw)은 흐릿하게밖에 못 본다.**
→ 이것이 ②·③이 반드시 필요한 이유를 강화한다.

**핸드오프**: ②에게 **"회전 계통, 방향 의존, 대략 이 정도 크기"라는 정량 가설**을, ③에게는
"이 정도 나올 것"이라는 기대치를 넘긴다.

## ①+④를 합치면 — 아는 것 / 모르는 것

**알게 됨:**
- 융합은 통과다(①).
- odom 드리프트의 **방향·대략 크기·회전vs병진·CW/CCW 비대칭** 그림(④).
- 우선순위 가설: 예) *"지배 오차는 회전 스케일 바이어스(미회전), ~N°/rev, 방향 의존 →
  스워브 조향각 캘리브레이션/트랙 기하 의심."*

**여전히 모름:**
- **진짜 크기** (④는 순환성으로 과소평가 + MCL 자체 cm·yaw 노이즈).
- **계통 vs 랜덤의 깨끗한 분리** (①은 재배치 불가, ④는 오염).
- **어느 물리 파라미터가 틀렸는지** 확정.

이 "모름"의 목록이 ②·③의 존재 이유이자 가치다.

## 그래서 ② UMBmark의 가치 — "정사각형은 yaw 오차 증폭기다"

- **작은 heading 오차로 정사각형을 돌면, 복귀점에서 큰 위치 오차로 증폭된다**(핀휠이 그것). 즉
  **square 경로 자체가 yaw 계통오차 증폭기**다.
- ①이 지목하고 ④가 흐릿하게만 본 그 **yaw 오차를, ②는 끝점 위치 오차라는 "줄자로 재기 쉬운 큰
  값"으로 바꿔** 잡는다.
- CW/CCW 5회씩의 무게중심 분석이 오차를 두 종류로 분해:
  - **Type A** (휠베이스 불확실 → CW/CCW 반대 방향)
  - **Type B** (스케일·방향 오차 → CW/CCW 같은 방향)

  이 A/B 버킷은 **①④가 이미 넘긴 CW/CCW 부호 패턴에 정확히 대응**한다. 즉 ②는 빈손이 아니라
  **가설을 들고 도착**해 확정·정량화만 한다.

**②의 산출물**: 실제 **보정계수** → 로봇에 반영해 odom 개선(루프 닫기). + 마크 재배치로 얻는
**랜덤오차 바닥값**(①④가 못 준 것).

---

## 그래서 ③ AprilTag의 가치 — "심판이자 전경로 진실"

- 독립·전경로·sub-cm 절대 진실. `scripts/run_evo.py`(ATE/RPE) 파이프라인이 여기서 제값을 한다.
- **②를 검증**: 보정계수가 맞았나? 끝점만 보는 ②가 놓친 **경로 중간 이탈**은 없나?
- **④를 검증**: 순환성·노이즈로 ④가 얼마나 과소평가했나? → **④의 편향 계수 산출**.

## 닫힘 루프

③가 "④는 진실 대비 이만큼 과소평가한다"를 한 번 재주면, **장비 없이 아무 데서나 도는 ④가
신뢰 가능한 상시 감시로 승격**된다. 비싼 일회성 진실(③)로 싼 상시 측정(④)을 **부트스트랩**한다.

```
①  회전이 범인 + 진실 필요        ┐
    │ (조준점)                    │  ← analyze_method1: 비폐합 / CW·CCW 리포트
    ▼                             │
④  방향·크기·회전vs병진 (하한)     │  ← analyze_method4 + evo(TUM)
    │ (정량 가설)                 │
    ▼                             │
②  계통/랜덤 분리 + 보정계수 ──► 로봇에 수정 반영
    │ (수정 + 검증대상)           │
    ▼                             │
③  전경로 절대 진실 (심판) ───────┘
    │
    └─► ④의 편향 보정 → ④를 "상시 감시"로 승격 (닫힘)
```

**한 줄 요약**: ①은 **질문을 좁히고**, ④는 **가설을 세우고**, ②는 **고치고**, ③은 **판결한 뒤
④를 재무장**시킨다. 분석·evo 인프라(`scripts/`, `recorder`)가 이 네 단계를 잇는 혈관이다.
