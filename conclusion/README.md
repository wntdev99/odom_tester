# conclusion — 테스트 결론 모음

> 각 측정 방법(①②③)의 **실주행 테스트에서 확실하게 확인된 결론**을 쌓아 두는 폴더.
> 모든 문서는 같은 구조를 따른다: **테스트 환경 → 테스트 방법 → 테스트 결과 → 결론**.
> 결론은 「✅ 확실하게 알게 된 것」과 「⚠️ 아직 모르는 것(경계)」을 반드시 분리한다.
>
> 상세 설계·명세는 [`../docs/`](../docs/), 진행 현황·TODO는 [`../todo.md`](../todo.md) 참조.

## 새 결론 문서 추가 규칙

1. 파일명: `method-<N>-conclusion.md` (예: `method-2-conclusion.md`).
2. 아래 4개 섹션 구조를 유지한다.
   - **1. 테스트 환경** — 로봇/토픽/프레임/EKF 구성 등 실측값.
   - **2. 테스트 방법** — 경로·조건(preset)·반복·튜닝·실제 실행 범위.
   - **3. 테스트 결과** — 관측된 수치·현상만 (해석 없이).
   - **4. 결론** — ✅ 확실히 알게 된 것 / ⚠️ 아직 모르는 것(경계) / ➡️ 다음 행동.
3. 추측·미검증은 반드시 「⚠️ 아직 모르는 것」으로 분리 (절대 정확도 주장 금지 — GT 있는 방법만).
4. 문서를 추가하면 위 **목록 표에 한 줄 추가**한다.
